1 Project Overview

MidFlix is a full-stack video streaming platform inspired by services like JioHotstar. It provides a complete content management system for administrators, a public-facing browsing experience, HLS video playback with adaptive quality, watch history tracking, and role-based access control.

Key Features

Content Management
Full CRUD for movies and series with seasons, episodes, genres, cast, and rich metadata. Supports image uploads and Mux video integration.
Public Browsing
No login required to browse, search, and watch content. Trending, new releases, genre filtering, and infinite scroll.
Video Streaming
HLS playback via Mux with adaptive quality selection, subtitle support, keyboard shortcuts, Picture-in-Picture, and fullscreen.
Watch History
Per-user progress tracking with auto-save every 15 seconds, resume playback, and a "Continue Watching" carousel for logged-in users.
Admin Dashboard
Premium admin panel with analytics, content stats, recent activity, top viewed content, and quick action shortcuts.
Role-Based Access
Sanctum token auth with Spatie permissions. Admin and subscriber roles with middleware-based route protection.

Access Model

Public-First Design: All content browsing and video playback is accessible without authentication (similar to JioHotstar). Login is only required for the admin dashboard and for features like watch history and continue watching.

10 Authentication & Authorization

10.1 Authentication Flow

User submits login form
↓
POST /api/v1/auth/login
↓
AuthService validates credentials
↓
Sanctum creates personal access token
↓
Token stored in localStorage (midflix-auth)
↓
Axios interceptor adds Authorization: Bearer header

10.2 Role-Based Access Control

RoleCapabilitiesProtected By
adminFull content CRUD, image upload, stats, season/episode managementauth:sanctum + role:admin middleware
subscriberSave/retrieve watch history, continue watching listauth:sanctum middleware
anonymousBrowse, search, view details, watch all contentNo middleware — fully public

10.3 Axios Interceptors

  • Request interceptor: Automatically adds Authorization: Bearer {token} from localStorage to every outgoing API request.
  • Response interceptor: On 401 errors, only redirects to /login if the user is on an /admin page — public browsing is never disrupted by auth errors.

11 Video Streaming & Playback

11.1 Mux Integration

Video content is hosted on Mux. Each content or episode stores a playback_id used to derive streaming and thumbnail URLs:

// HLS streaming URL
https://stream.mux.com/{playback_id}.m3u8

// Thumbnail image URL
https://image.mux.com/{playback_id}/thumbnail.jpg

11.2 Watch History Tracking

TriggerBehavior
Every 15 secondsAuto-save progress via React Query mutation (throttled by ref timestamp)
On video pauseForce-save current position immediately
On video endedSave progress = duration. Auto-navigate to next episode if series.
On page unloadbeforeunload handler uses fetch with keepalive: true for reliable delivery
Completion threshold: A video is marked "completed" when the viewer reaches 90% of the total duration. Completed items are excluded from the "Continue Watching" carousel on the homepage.

14 Getting Started

14.1 Prerequisites

  • Docker and Docker Compose installed
  • Node.js 22 (via NVM) for frontend development
  • PHP 8.2+ and Composer (if running backend outside Docker)

14.2 Quick Start

# Clone the repository
git clone <repository-url>
cd MidFlix

# Start all backend services
docker-compose up -d

# Run database migrations and seed default data
docker-compose exec app php artisan migrate --seed

# Start the frontend development server
cd frontend
nvm use 22
npm install
npm run dev

14.3 Access Points
